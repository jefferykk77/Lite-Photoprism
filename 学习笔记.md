# 数据库与 sqlx 学习笔记

## 1. 核心概念与原理

### 1. CRUD 对应的 SQL 语句
| 操作 | SQL | 含义 |
|:---|:---|:---|
| **C**reate | `INSERT INTO photos (name) VALUES (?)` | 插入一条记录 |
| **R**ead | `SELECT * FROM photos WHERE id = ?` | 查询记录 |
| **U**pdate | `UPDATE photos SET name = ? WHERE id = ?` | 修改记录 |
| **D**elete | `DELETE FROM photos WHERE id = ?` | 删除记录 |

### 2. 关键方法记忆
- `db.Get(&单个对象, SQL, 参数...)` → 查一条
- `db.Select(&切片, SQL, 参数...)` → 查多条
- `db.MustExec(SQL, 参数...)` → 执行写操作（插入 / 更新 / 删除）

### 3. 关于 `db` Tag 的底层逻辑
`// db tag 告诉 sqlx 这个字段对应数据库的哪一列`

**认知对齐**：你的理解非常精准。`sqlx` 是建立在 Go 标准库 (`database/sql`) 之上的工具。Go 生态维持了一种默契：大家都认 `db` 这个标签。这样一来，无论是在 Go 之上建立的哪种 SQL 衍生工具，大家都能识别同一个 `db:"xxx"`，从而维持了生态体系的高度统一。只要标对了这个 tag，`sqlx` 就像插头对准了插座，完美识别字段。

## 2. 实战踩坑与正确心法

### 动词的本质
SQL 世界里没有 `GET` 这个动词。你要向数据库**“查询”**，动词永远是 `SELECT`。
- 正确语句：`"SELECT * FROM Photos WHERE id=?"`
- *(类比：从(FROM) Photos表里，选择(SELECT)全部列(*)，当(WHERE) ID等于某值时。)*

### 1. 怎么使用 Schema？
在对数据库做任何事之前，先得把表建好。
- 调用你写的初始化函数： `schema := repository.InitDB()`
- 扔给数据库执行： `db.MustExec(schema)`
- *(注：建表是 `CREATE TABLE`，自增是 `AUTOINCREMENT`。)*

### 2. 连接参数
- 错误示范：`Connect("sqlite", "sqlite")`
- 正确：`Connect("sqlite", "test.db")`（驱动名是固定的，你的文件名可以自己取，但要有后缀方便识别）。

### 3. 如何手写 INSERT（插入动作）
`MustExec` 接收的是纯正的 SQL。不要带 `WHERE`。
- **骨架**：`db.MustExec("INSERT INTO 表名 (列1, 列2) VALUES (?, ?)", 值1, 值2)`
- **比如**：`db.MustExec("INSERT INTO Photos (path, content) VALUES (?, ?)", "/img/1.jpg", "hello")`

### 4. 如何手写 GET（查询动作）
`Get` 的第一参数是目标容器的**内存地址**。
- **骨架**：`db.Get(&你的变量, "SELECT * FROM 表名 WHERE 某条件 = ?", 条件值)`
- **比如**：`db.Get(&photo, "SELECT * FROM Photos WHERE id = ?", 1)`

### 5. SQL 关键字与表名严谨性
SQL 对拼写极其敏感，不能受口语习惯影响。
- **关键字踩坑**：插入数据的值是 `VALUES` (复数)，不要误写为 `VALUE`。
- **大小写与复数对齐**：使用 `INSERT INTO Photos`，表名必须与 `db_init.go` 初始化建表时的名称（包含后缀 s）做到字符级对齐。

## 3. 常问答疑 (FAQ)

### 1. SQL 全称是什么？
**S**tructured **Q**uery **L**anguage (结构化查询语言)。

### 2. Exec 和 MustExec 有什么本质区别？
- `Exec`：执行 SQL，如果出错，**温和地返回一个 错误对象 (`err`)**，交由外部通过 `if err != nil` 判断并处理。
- `MustExec`：执行 SQL，如果出错，**当场崩溃退出程序 (Panic)**。只适合用于“必须成功，否则没法继续工作”的场景（如系统启动时的建表初始化）。

### 3. 连接数据库为什么要多包一层 sqlx，不直接用 sqlite？
- **类比**：`sqlite`（驱动）是底层的“汽车引擎”，只懂最基础的物理运转；`sqlx` 是“方向盘和中控台”。如果没有 `sqlx` 的自动化装箱功能（基于 `db` 标签把数据塞进 Go 结构体），你就只能使用底层库手写大量繁琐的代码来逐个读取和赋值拼装数据。

### 4. 为什么 Repository 层不建议直接使用 log.Println？
- **第一性原理**：Repo 层（库管员）职责极度单一，它只是命令执行者。当遇到错误时，它的标准行为是**把错误原封不动地抛给上一级**（即 `return err`），而不是自己在大堂里大喊大叫（直接打印日志）。
- **本质隔离**：究竟是打印在终端，写进文件，还是通过网络返回 `HTTP 500` 报错给用户？这种业务决策，应该永远由在最顶层的 `main` 或 `Handler（服务员）` 去决定。
